How Check works

Check does not extract a secret. It runs a layered steganalysis pipeline and scores how strongly the video looks like it contains hidden data.

  1. Signature scan — known stego markers (e.g. VSTG).
  2. Container structure — appended trailers after media.
  3. vsteg active probe — confirm vsteg append / LSB / DCT payloads.
  4. MP4 forensics — unreferenced mdat slack vs stsz totals (OpenPuff-style).
  5. FFmpeg / ffprobe — codec, streams, metadata consistency.
  6. LSB / RS statistics — chi-square, sample-pair, Regular–Singular analysis.
  7. DCT mid-band — QIM-like coefficient clustering.
  8. Keyframe DCT anomaly — StegoForge-style I-frame mid-band z-scores.
  9. ML ensemble — optional RandomForest over those features (pip install -e ".[ml]").

Score guide: 0–24 clean · 25–59 suspicious · 60–100 likely-stego. ffprobe notes with +0 weight are informational only.

How Compare works

Compare places two videos side by side using ffprobe attributes, then samples frames to measure visual drift.

  1. Attributes — codec, resolution, bitrate, fps, duration, streams, metadata.
  2. Graphs — scalar metrics, stream counts, per-frame difference, luma histograms.
  3. Frame sampling — mean absolute difference on grayscale frames (when resolutions match).
